# 115finance_engine: 高效能財務數據處理與互動引擎 (C-based)

##  專案動機
身為會計背景的學生，在處理大量財務分錄時常發現傳統手動整理的低效。本專案旨在利用 C 語言的效能優勢，建立一個能快速處理、分類並排序大規模交易紀錄的引擎。這不僅解決了實務上的痛點，也展現了我將商科知識與 computer science 結合的能力。

##  核心功能
- **互動式終端機介面 (Interactive CLI)**：提供使用者友善的選單系統，可即時切換不同的數據處理與檢視模式。
- **多維度高效能排序**：實作多個比較函數（Comparator Functions），結合時間複雜度 $O(n \log n)$ 的 `qsort` 演算法，允許使用者按「金額 (高至低)」、「日期 (新至舊)」或「會計科目 (A to Z)」動態重新排列數據。
- **CSV 自動化解析**：讀取標準財務數據格式，將非結構化文本精準轉為高效的 `struct` 結構體。
- **動態記憶體管理**：透過 `malloc` 與 `free` 在 Heap 區動態配置與精確控制系統資源，避免處理大規模數據時出現 Memory Leak。

##  技術細節
- **開發語言**：C11
- **核心資料結構**：Structured Arrays, Dynamic Memory (Heap)
- **進階指標應用**：使用 Function Pointers (函數指標) 驅動多重排序邏輯。
- **字串處理與分析**：使用 `strtok` 進行數據清洗，並透過 `strcmp` 進行時序與類別的比對。
- **實作動態擴容機制**：為了應對無法預估大小的巨量 CSV 檔案，本專案未使用寫死的靜態陣列，而是結合 malloc 與 realloc 實作了記憶體動態擴充。當資料量突破當前限制時，系統會自動以 $O(n)$ 的時間成本將容量翻倍，使整體的資料插入時間複雜度攤平後仍保持極高效的 $O(1)$ Amortized Time，展現對記憶體底層與演算法效能的深度考量。

##  檔案說明
- `finance_engine.c`: 專案主程式原始碼。
- `data.csv`: 測試數據檔案（包含 100 筆模擬日常開銷的 ID、日期、類別、金額與備註）。

##  技術筆記：CSV 解析原理
本專案在不依賴外部函式庫的情況下，原生實作了 CSV 檔案的讀取與記憶體映射：
1. **`fopen`**：建立檔案指標，安全地載入數據。
2. **`fgets`**：逐行讀取文本，避免一次性載入過大檔案導致記憶體溢位。
3. **`strtok`**：核心字串切割邏輯，以逗號 `,` 為定界符，將單行文本拆解並轉換型態（如 `atoi`, `atof`），最終安全地存入自定義的 Transaction 結構體中。
