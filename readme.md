# 115finance_engine: 高效能財務數據處理與自動化匯出引擎

## 📌 專案動機
身為具備會計背景的學生，在處理海量財務分錄時，我深刻感受到傳統辦公軟體（如 Excel）在面對非結構化數據與大規模運算時的效能瓶頸。為了徹底解決自動化對帳與數據分析的實務痛點，我決定跳脫「軟體使用者」的框架，以 **C 語言** 獨立開發這套高效能財務數據處理引擎。

本專案的核心目標不只是功能實作，更是對計算機科學底層邏輯的嚴謹挑戰：
- **挑戰效能極限**：捨棄高階語言的便利，選擇 C 語言以達成對記憶體的精確控制。透過實作 **$O(n \log n)$ 的Quick Sort**，確保系統在處理百萬級距的財務數據時，依然保有極致的運算效率。
- **實作動態擴容機制**：為了應對未知規模的數據流，我捨棄了靜態陣列，自主實作了**動態記憶體擴張邏輯**。透過 `realloc` 策略優化空間利用率，展現對資料結構與記憶體管理的深度思考。
- **跨領域知識融合**：利用對會計科目與財務邏輯的理解，將專業知識轉化為程式邏輯，實現了具備「互動式選單」與「多維度排序」的解析引擎。

## 🚀 核心功能
- **互動式終端機介面**：提供使用者友善的選單系統，可即時切換不同的數據處理與檢視模式。
- **多維度高效能排序**：實作多個 Comparator Functions，結合時間複雜度 $O(n \log n)$ 的 `qsort` 演算法，允許使用者按「金額」、「日期」或「會計科目」動態重新排列數據。
- **自動化財務摘要**：整合數據聚合邏輯，能自動按會計科目進行金額加總，生成類別財務報表。
- **資料持久化匯出**：支援將**當前排序結果**匯出為標準 CSV 格式，實現與外部報表系統（如 Excel）的無縫對接。
- **動態記憶體管理**：透過 `malloc` 與 `free` 在 Heap 區動態配置資源，確保在大規模運算下依然保有高穩定性與零洩漏（Zero Leak）。

## 🛠️ 技術細節
- **開發語言**：C11
- **專業架構設計**：採用標頭檔與實作分離的模組化設計，提升程式碼的可維護性。
- **進階指標應用**：使用 **Function Pointers** 驅動多重排序邏輯，增加系統的可擴充性。
- **Amortized Time 分析**：採用翻倍擴張策略（Doubling strategy），使資料插入的時間複雜度在平攤後仍保持極高效的 **$O(1)$ Amortized Time**。

## 📂 檔案說明
```text
115finance_engine/
├── include/              # 標頭檔定義 (.h)
│   └── finance_engine.h
├── src/                  # 原始碼實作 (.c)
│   ├── finance_engine.c
│   └── main.c
├── data/                 # 財務數據檔案 (.csv)
│   └── data.csv
├── .gitignore            # 版本控制忽略規範
└── README.md
```
- `main.c`: 處理使用者互動選單與檔案讀取流。
- `finance_engine.c`: 核心演算法實作（排序、匯出、數據摘要）。
- `finance_engine.h`: 結構體定義與函式介面宣告。
- `data.csv`: 測試數據檔案（包含模擬日常開銷紀錄）。
- `sorted_export.csv`: (執行選單選項 4 後產生) 經由引擎處理並匯出的結構化數據。

## 📝 技術筆記：系統架構設計與 Git 管理
1. **運算與存取功能拆分**：為了保有系統的彈性，我將**運算與存取**功能拆分。使用者可先進行預覽，確認無誤後再執行資料持久化，以避免頻繁的磁碟 I/O，優化系統效能。
2. **防禦性編程**：在讀取與動態擴容過程中皆實作了指標安全性檢查，確保在異常情況下能安全釋放資源。
3. **版本控制規範**：透過 `.gitignore` 嚴格過濾編譯產生的執行檔與動態生成的結果檔，僅保留核心程式碼與必要數據，體現版本管理意識。
