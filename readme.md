# 115finance_engine: 高效能財務數據處理引擎 (C-based)

## 📌 專案動機
身為會計背景的學生，在處理大量財務分錄時常發現傳統手動整理的低效。本專案旨在利用 C 語言的效能優勢，建立一個能快速處理、分類並排序大規模交易紀錄的引擎。這不僅解決了實務上的痛點，也展現了我將商科知識與計算機科學結合的能力。

## 🚀 核心功能
- **CSV 自動化讀取**：解析標準財務數據格式，將非結構化文本轉為高效的 `struct` 結構。
- **高效能排序**：實作 `qsort` (Quick Sort)，確保在處理萬級數據時仍保有 $O(n \log n)$ 的運算效率。
- **動態記憶體管理**：透過 `malloc` 與 `free` 精確控制系統資源，避免處理大規模數據時出現 Memory Leak。
- **數據分析報告**：自動計算總支出與平均消費，並依金額權重輸出報告。

## 🛠️ 技術細節
- **開發語言**：C11
- **核心資料結構**：Structured Arrays, Dynamic Memory (Heap)
- **演算法**：Quick Sort ($O(n \log n)$)
- **字串處理**：使用 `strtok` 進行數據清洗與模式匹配。

## 📂 檔案說明
- `finance_engine.c`: 主程式邏輯。
- `data.csv`: 測試數據檔案（包含日期、類別、金額、備註）。

## 📈 執行方式
1. 確保電腦已安裝 C 編譯器 (如 GCC)。
2. 將 `data.csv` 置於與執行檔相同的路徑下。
3. 編譯並執行：
   ```bash
   gcc finance_engine.c -o finance_engine
   ./finance_engine
---

### 二、 關於 CSV 如何讀入 `finance_engine.c`

在 C 語言中，讀取 CSV 並非「一鍵完成」，而是透過一系列精密的邏輯將文字拆解。你在程式中已經包含了這些邏輯，這裡幫你釐清它的運作流程：

1.  **檔案放置位置（最重要）**：
    * 你的 `data.csv` 必須放在與 `finance_engine.exe`（編譯後的執行檔）**同一個資料夾**。
    * 如果你在 VS Code 裡點擊「執行」，請確保 VS Code 下方的終端機路徑（Terminal Path）正處於你的專案資料夾。
2.  **`fopen` 開啟檔案**：程式會透過檔案指標去硬碟尋找這個檔案。
3.  **`fgets` 逐行讀取**：程式一次只讀一整行文字（包含逗號）。
4.  **`strtok` 切割字串**：這是關鍵！它會尋找「逗號」的位置，把一長串文字切成「ID」、「日期」、「金額」等小塊。
